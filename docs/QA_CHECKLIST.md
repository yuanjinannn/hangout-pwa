# QA 冒烟测试清单

## 启动

```powershell
.\start-pwa.cmd
```

打开：

```text
http://127.0.0.1:4173
```

## 自动化检查

```powershell
.\scripts\smoke-ui.cmd
```

脚本会自动启动本地服务，并覆盖地图搜索或本地兜底、活动创建、投票、确认、举报、重置演示数据和 PWA 缓存。

## 主流程

1. 首页
   - 页面能加载出“约玩票夹”。
   - 底部导航包含：票夹、卡库、发起、卡面。
   - 刷新页面后仍能正常恢复。

2. 地点卡库
   - 进入“卡库”。
   - 输入“咖啡”或“火锅”。
   - 点击“搜地图”。
   - 预期：配置 `AMAP_KEY` 时地图搜索返回地点，状态提示导入成功，列表出现新地点。
   - 预期：未配置 `AMAP_KEY` 时提示使用本地推荐，不阻断后续创建活动。

3. 创建活动票
   - 进入“发起”。
   - 输入标题、邀请名单、人数上限、备注。
   - 切换私密/公开。
   - 点击“创建活动票”。
   - 预期：自动进入卡面，新活动排在当前详情。

4. 投票和确认
   - 在活动卡面点击收藏或加入。
   - 对一个时间槽选择“有空/也许/没空”。
   - 点击“封存最佳方案”。
   - 预期：活动状态变为“已确认”，建议时间和建议地点有值。

5. 举报审核
   - 对公开活动点击“举报这个公开活动”三次。
   - 预期：活动状态变为“待审核”，不再作为正常公开招募活动出现。

6. PWA
   - 浏览器支持时出现安装入口。
   - 首次加载后存在基础离线缓存。
   - 更新 `sw.js` 的缓存名后，旧缓存会被清理。

7. 演示数据重置
   - 点击右上角身份头像。
   - 点击“重置演示数据”。
   - 取消确认时，当前数据保持不变。
   - 确认重置时，回到初始票夹、初始地点和默认身份。

## 回归关注点

- 中文输入法输入时不要丢字、跳光标。
- 移动端 390px 宽度下按钮文字不溢出。
- 地图接口失败时保留本地推荐，不阻断用户创建活动。
- 邀请链接导入失败时有可理解提示。
- localStorage 数据损坏时能自动回到默认状态。
